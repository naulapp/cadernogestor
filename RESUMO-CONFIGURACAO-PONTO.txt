--- Para que serve o SESSION_SECRET? ------------------------------------------

É uma senha SÓ do Worker. Depois que o funcionário faz login com PIN, o Worker
devolve um "token de sessão" (JWT). O SESSION_SECRET é usado para ASSINAR e
VERIFICAR esse token. Sem ele, login/upload/registro não funcionam de forma
segura. Não é Firebase, não é JSON — é uma string longa aleatória que você
colou no painel da Cloudflare como Secret.

[ ] C) CLOUDFLARE — Bucket R2
    Nome do bucket: cadernogestor-ponto  ← ESTÁ CERTO se for esse o nome que
    você criou no painel R2. Tem que ser IGUAL ao bucket_name no arquivo:
    worker/wrangler.toml  (linha bucket_name = "cadernogestor-ponto")
    Se no R2 o bucket tiver OUTRO nome, altere o wrangler.toml e faça deploy
    de novo.

[ ] D) CLOUDFLARE — Worker (GitHub naulapp/cadernogestor)
    - Repositório conectado; diretório raiz da build: /worker
    - Comando de implantação: npx wrangler deploy (ou o que a Cloudflare
      mostrar como padrão)
    - worker/wrangler.toml com name = "cadernogestor" (igual ao nome do Worker
      no painel)
    - Última build: SUCESSO (não "Falha na última build")
    - Domínio workers.dev: HABILITADO (para aparecer URL pública)
    - URL anotada (ex.: https://cadernogestor.<sua-conta>.workers.dev)

[ ] E) CLOUDFLARE — Secrets do Worker (Configurações → Variáveis e segredos)
    - FIREBASE_SERVICE_ACCOUNT = JSON inteiro da conta de serviço (uma linha

    - SESSION_SECRET = string longa aleatória (ver explicação no topo)

    PONTO_BUCKET → bucket cadernogestor-ponto
    (Se o deploy ler o toml certo, o painel pode mostrar o binding após build
    com sucesso.)

CONFERÊNCIA RÁPIDA (testes)
================================================================================

1) No navegador, abra:  https://SEU-WORKER.workers.dev/health
   Deve retornar JSON com "ok": true e "service": "cadernogestor" (ou similar).

2) Se /health falhar: build falhou, secrets faltando, ou URL errada.

3) Se login no PWA falhar: FIREBASE_SERVICE_ACCOUNT, SESSION_SECRET, ou
   código da empresa / PIN errados.

4) Bucket com nome diferente de cadernogestor-ponto: altere worker/wrangler.toml
   bucket_name, commit, espere novo deploy.
